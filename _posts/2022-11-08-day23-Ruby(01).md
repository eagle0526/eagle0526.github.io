gem
------

gem 版本介紹、和bundle有什麼不一樣、Gemfile.lock是用來做啥的  

### gem 版本代表的意思
這樣設計的原因是，如果今天加小物件，跳 patch 版號，如果今天是加新功能，跳 minor 版號，如果大更新，跳 major 版號

```md
> gem 'xxx' 3.1.7
> 3 => major => 完全不同的產品
> 1 => minor => 有可能會壞掉  
>  7 => patch => 增加不太重要的功能 （可以隨便更新來用）
>
> gem 'xxx', '~> 3.1.7' 3.2.0 不會裝
> gem 'xxx', '>3.1.7' 3.2.0 會裝
```

> --  
> ~的意思是，會去裝比較安全的版本，所以今天如果改動的是 patch 就會安裝，跳 minor 版號就不會安裝  
> --  
> {: .block-tip}

### gem vs bundle

1. gem 是一次裝一個指定的套件  
2. bundle 是一次下載寫在 gemfile 的描述檔  
```md
> (1) 找 gemfile  
> (2) gem install ...  
> (3) 產生 Gemfile.lock  
```
  
### Gemfile.lock 是啥
Gemfile.lock => 會產生一個“多個檔案相依性”的檔案，這可以解決版本不同的相依性問題  
假設今天 A 套件需要 B 套件 2.0 版本  
C 套件需要 B 套件 3.0 版本  
bundle 完後，可以解決上面這個問題

p、puts 的差異
------

先創一個陣列
```md
> lst = [1,2,3]
```

### p
1. p會保留要印出來的殼，並印在一行  
2. 會有回傳值  
```md
> p lst        # [1,2,3]
```

### puts
1. puts 會整個拆掉，並逐行印  
2. 沒有回傳值  
```md
> puts lst     # 1
>                2
>                3
```

錯誤訊息
------

### JS
JS中，抓取錯誤訊息 - catch
```md
> function hi() {
>     // Exception
>     throw "aaaa"               -> throw是把一個錯誤丟出來
> }
> 
> try {                          -> 嘗試執行下面的fc
>     hi()
>     catch (e) {                -> 如果有錯，做下面的事情
>         console.log("error")
>     }
> }
```

### Ruby
Ruby中，抓取錯誤訊息 - rescue

```md
> def bmi_calc(w, h)
>   w / h                     -> 不要把錯誤判斷寫在這，因為讓其他人來幫忙判斷是否錯誤
> end
> 
> begin
>   puts bmi_calc(180, 70)
> rescue                      -> 救援功能，如果輸入值有問題，執行下面那一段
>   puts "error"
> end
```
  
  
rescue簡化寫法
```md
> def bmi_calc(w, h)
>   w / h                     -> 不要把錯誤判斷寫在這，因為讓其他人來幫忙判斷是否錯誤
> rescue => exception
>   puts "error"
> end
```

擴充功能
------

### JS
  
JS用prototype可以幫物件、陣列、字串...加功能  
```md
> String.prototype.is_valid = 123
> 
> console.log("a03456".is_valid)                  # 123
```

### ruby
  
使用class可以幫任何物件家功能  
```md
> class String
>   def is_valid
>     p  123
>   end
> end
> 
> p "a03456".isis_valid                           # 123
```
